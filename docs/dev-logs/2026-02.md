# 🚀 开发日志 | 2026年02月

## 📈 项目概览

- **当前阶段**: 测验管理模块（Test）CRUD 闭环建设
- **核心目标**: 从试卷创建到题目维护、班级发布、答题卡下载，形成可演示的完整链路。

---

## 📅 2026-02-01 | 测验 CRUD 起步（第1天）

### ✅ 已完成

- [✓] 后端新增测验基础接口：列表、新建、编辑、删除。
- [✓] 前端在仪表盘增加测验路由入口，完成页面骨架挂载。
- [✓] 打通测验列表展示字段：`quiz_id/title/start_time/status`。

### 🛠 遇到问题

- **问题**: 测验数据结构与页面展示字段命名不一致，前端渲染报空。
- **解决**:
  1.  在前端映射层统一转换（`quiz_id -> id`，`title -> testName`）。
  2.  固定状态默认值为 `草稿`，避免空状态导致模板异常。

---

## 📅 2026-02-02 | 题目 CRUD 并入测验（第2天）

### ✅ 已完成

- [✓] 完成题目增删改查，支持 `单选/多选/判断/主观` 四种题型。
- [✓] 建立测验题目模型：题目直接归属 `quiz_id`，并新增测验内序号 `sequence_number`。
- [✓] 在前端实现题目编辑弹窗，支持不同题型的动态字段与答案录入。

### 🛠 遇到问题

- **问题**: 题号与数据库主键混用，编辑/删除时目标记录不稳定。
- **解决**:
  1.  明确 `question_id` 仅用于 API 精准定位。
  2.  明确 `sequence_number` 仅用于测验内展示与排序。
  3.  增加同测验唯一约束，防止重复题号。

---

## 📅 2026-02-03 | 发布与关联 CRUD（第3天）

### ✅ 已完成

- [✓] 完成测验发布到班级能力，支持多班级一次性分配。
- [✓] 新增发布关联管理：可查询当前发布列表并取消单条发布。
- [✓] 前端发布弹窗支持多选班级并展示发布结果数量。

### 🛠 遇到问题

- **问题**: 重复发布同一班级会出现冗余数据。
- **解决**:
  1.  后端发布前增加存在性检查，已存在则跳过。
  2.  返回 `created_count`，前端准确提示本次新增数量。

---

## 📅 2026-02-04 | 答题卡生成下载链路（第4天）

### ✅ 已完成

- [✓] 后端打通“测验 -> JSON 参数 -> Python 生成答题卡”的调用链路。
- [✓] 支持下载 `PDF/PNG`，并统一文件名规则为 `答题卡_{测验名}`。
- [✓] 增加生成参数落盘到 `backend-ocr/rejson/`，便于复现和排查。

### 🛠 遇到问题

- **问题**: 出现“文件损坏/扩展名正确但打不开”的下载问题。
- **解决**:
  1.  后端改为先读取二进制内容再响应，避免临时文件时序问题。
  2.  增加 PNG/PDF 文件签名校验，异常时直接返回错误。
  3.  前端优先解析 `Content-Disposition`（含 `filename*`）确保中文文件名稳定。

---

## 📅 2026-02-05 | 稳定性收敛与 ZIP 下载（第5天）

### ✅ 已完成

- [✓] 优化下载入口：由下拉菜单改为单按钮，默认下载 ZIP。
- [✓] 完成 ZIP 打包内容：`JSON + PDF + PNG` 三件套一次下载。
- [✓] 修复子进程输出解析不稳定问题，引入标准输出标记 `OUTPUT_PATH=`。
- [✓] 修复 `Malformed UTF-8` 异常，统一清洗脚本输出再写入 JSON 错误响应。

### 🛠 遇到问题

- **问题**: 脚本成功执行但后端偶发“未找到输出文件”。
- **解决**:
  1.  不再依赖中文日志行解析路径，优先读取 `OUTPUT_PATH=`。
  2.  生成失败时输出完整清洗后的诊断信息，便于定位环境差异。
