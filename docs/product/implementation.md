# MVP 实施清单

## 1. MVP 边界

仅覆盖核心闭环：
建卷 -> 生成答题卡 -> 上传 -> OCR/判分 -> 教师复核 -> 成绩查看

不包含：多租户、复杂权限、高级可视化看板。

## 2. 必做功能

### A. 认证与教师资料
- 注册/登录/退出
- 忘记密码
- 资料编辑

### B. 班级与学生
- 班级 CRUD
- 学生 CRUD
- CSV 批量导入

### C. 题库与试卷
- 题目 CRUD（单选/多选/判断/主观）
- 试卷创建与题序分值配置
- 状态流转（草稿/发布/结束）
- 发布到班级

### D. 答题卡与提交
- 生成答题卡 PNG
- 单张/批量上传
- 保存 submission 与原图路径

### E. 批改流程
- Python OCR 识别
- 客观题自动判分
- 主观题 AI 建议分
- 教师复核与最终提交

### F. 成绩与导出
- 按试卷/班级/学生查询
- 基础统计（均分、及格率、最高/最低）
- Excel 导出

## 3. 暂缓项

- 跨班级高级对比面板
- 移动端完整管理功能
- 主观题自动最终判分

## 4. 风险与对策

- 图片质量差：上传提示 + 失败重传
- 客观题误判：低置信度标记人工复核
- AI 波动：仅作为建议分
- 跨服务超时：重试与幂等键（submission_id）

## 5. 验收标准

- 可跑通完整业务闭环
- 客观题准确率 >= 95%（样例集）
- 总分等于明细分之和
- 批量提交可稳定处理并支持重试

## 6. 推荐排期（4 周）

1. 第1周：数据库与基础 CRUD
2. 第2周：答题卡生成与上传链路
3. 第3周：OCR/判分与复核页面
4. 第4周：成绩导出与稳定性修复

## 7. 数据库优先级

- P0：`users/teachers/classes/students/questions/quizzes/quiz_question_items/quiz_assignments/submissions/answer_details`
- P1：补索引与唯一约束
- P2：处理状态字段与可观测性增强
