# 产品概述

## 1. 项目背景

本系统旨在帮助中学教师个人高效管理试卷、生成答题卡，并通过上传学生填涂后的答题卡图片实现自动/半自动批改。系统针对教师单个学科使用，支持线下纸质答题卡与线上数字化批改的混合模式。客观题通过 OCR 自动判分，主观题使用 AI 大模型生成建议分数，由教师最终复核确认。同时提供班级及学生成绩的多维度统计分析。

### 目标用户

中学教师（单用户模式，无需多用户协作，无学生端登录）

### 核心特点

- 生成独立电子答题卡图片（PNG/JPG），教师可插入 Word 试卷中打印
- 学生纸质填涂答题卡后，教师/学生用手机或电脑拍照上传
- 客观题自动判分，主观题 OCR 提取文字 + AI 建议分数 + 教师复核
- 支持班级成绩统计、学生多试卷平均分分析等

## 2. 项目目标

### 功能目标

实现试卷从创建、答题卡生成、批改到成绩分析的全流程

### 技术目标

前后端分离，轻量可维护；集成 OCR 和 AI 大模型辅助批改

### 用户体验目标

极简浅色风格，操作流畅；移动端重点支持拍照上传

## 3. 假设与约束

### 假设

- 仅一名教师使用
- 学生信息由教师手动维护
- AI/OCR 准确率非 100%，需教师复核最终得分

### 约束

- 无学校级多用户/多学科支持
- 不涉及学生端登录
- 数据存储在 MySQL，图片文件存储在服务器本地或对象存储

## 4. 核心功能模块

### 4.1 用户管理模块（教师个人）

- **教师登录/注册**：用户名 + 密码登录；支持忘记密码（邮箱重置）
- **个人信息管理**：查看/编辑姓名、学科、性别、电话、邮箱；修改密码

### 4.2 班级与学生管理模块

- **班级管理**：创建/编辑/删除班级（班级号、入学年、毕业年、是否毕业）
- **学生管理**：增删改查学生信息（学号、姓名、性别、年龄、头像图片）；支持手动添加或简单批量导入

### 4.3 题库管理模块

- **题目创建/管理**：支持单选、多选、判断、主观四种题型
- **题目字段**：内容、选项（JSON）、正确答案、解析

### 4.4 试卷管理模块

- **试卷创建**：设置试卷标题、题号列表、每题分数、排序；直接生成电子答题卡图片（PNG/JPG）
- **试卷编辑/删除**：草稿状态下可编辑
- **试卷状态管理**：草稿 / 已发布 / 已结束 / 已归档
- **向班级发布试卷**：一份试卷可发布给多个班级（仅记录关联关系）

### 4.5 答题卡生成与批改模块

#### 答题卡生成

基于试卷设置自动生成图片，包含：

- 个人信息填写区（班级、学号、姓名）
- 客观题填涂区（泡泡/方格）
- 主观题答题空白区
- 定位点（用于图像矫正与区域定位）

#### 答题卡上传

- 支持 PC 网页上传或 uni-app 移动端（相机/相册）上传
- 支持批量上传

#### 智能批改流程

1. **客观题自动判分**：使用 Python OCR 识别填涂情况，比对正确答案计算得分
2. **主观题处理**：OCR 提取手写/印刷文字 → 调用 AI 大模型生成建议分数 → 存入数据库
3. **教师批改**：查看 OCR 提取结果 + AI 建议分数 → 确认或修改每题最终得分
4. **提交记录**：保存答题卡图片路径、每题得分、总分、批改状态

### 4.6 成绩查看与分析模块

#### 成绩查看

- 按试卷、按学生、按班级查看总分及每题得分
- 支持导出 Excel

#### 学生个人分析

- 查看该学生多份试卷的平均分、错误率、成绩折线图

#### 班级分析

- 多份试卷的班级平均分、题均得分、得分分布直方图

#### 跨班级对比

- 单份试卷在不同班级的平均分、题均得分分布对比

### 4.7 系统级功能

- 全局搜索：支持试卷、学生、题目快速搜索
- 数据分页与加载优化

## 5. 交互流程

### 完整业务流程

1. **教师登录** → 试卷管理
2. **创建试卷**：设置题号、分值、排序
3. **生成答题卡图片** → 下载并插入 Word 试卷打印
4. **发布到班级**
5. **学生纸质作答**
6. **教师/学生拍照上传答题卡**
7. **系统 OCR 处理**：客观题自动判分 → 主观题 AI 建议分数
8. **教师复核修改** → 出分
9. **成绩分析**：查看统计图表与数据

### 班级管理流程

1. **班级管理** → 创建班级
2. **添加学生**（含头像上传）

## 6. 技术实现要求

### 6.1 前端技术栈

- **PC 网页**：Vue 3 + DHTMLX JS Library（极简风格，浅色模式）
- **移动端**：uni-app（主要实现拍照上传、查看成绩）

### 6.2 后端技术栈

- **主后端**：Laravel 11（数据管理、API、文件上传、认证）
- **辅助处理**：Python 3.9+（答题卡生成、OCR 识别、AI 大模型调用）

### 6.3 数据库

- MySQL 8.0+（字符集 utf8mb4_unicode_ci）

### 6.4 部署

- 云服务器或本地部署
